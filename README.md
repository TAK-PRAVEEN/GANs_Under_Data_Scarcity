# GANs Under Data Scarcity

### An Empirical Study of GAN Loss Functions When Training Data Is Limited

> **📄 Research Paper (PDF):** _[Google Drive link »](PASTE_YOUR_GOOGLE_DRIVE_PDF_LINK_HERE)_
---

## Abstract

Generative Adversarial Networks (GANs) are notoriously sensitive to the amount of
available training data and to the choice of adversarial objective. This project
presents a **controlled empirical study** of how four widely-used GAN loss formulations
behave as the training set is progressively starved of data. Using a fixed **DCGAN**
backbone on **Tiny-ImageNet-200** (64×64 RGB), we train a full grid of
**4 loss functions × 4 dataset sizes (16 runs total)** under identical architecture,
optimizer, and schedule, and analyse them along three axes: **convergence behaviour**,
**training stability**, and **computational cost**. Our findings show that no single loss
dominates across all regimes — the “best” objective depends heavily on how scarce the
data is and on the available compute budget.

---

## Research Questions

1. **Q1 — Stability vs. data size.** How does each adversarial loss respond as the
   training set shrinks from 25,000 down to 1,000 images?
2. **Q2 — Loss robustness.** Which objectives degrade gracefully under scarcity, and
   which collapse catastrophically?
3. **Q3 — Cost.** What is the compute trade-off (epoch time) of the more “stable”
   objectives, and is it justified in the low-data regime?

## Experimental Setup

| Component | Choice |
|---|---|
| **Backbone** | DCGAN (transposed-conv Generator / strided-conv Discriminator) |
| **Resolution** | 64 × 64 × 3 (Tanh output, inputs normalized to [-1, 1]) |
| **Latent dim** | `z_dim = 100` |
| **Generator params** | ≈ 3.58 M |
| **Discriminator params** | ≈ 2.77 M |
| **Optimizer** | Adam, `lr = 2e-4`, `betas = (0.5, 0.999)` |
| **Batch size** | 32 |
| **Epochs** | 50 per run |
| **Critic steps** | `n_critic = 5` for WGAN / WGAN-GP, else 1 |
| **Augmentation** | Random horizontal flip |
| **Seed** | 42 (deterministic class-balanced sampling) |

### Dataset & Splits

The study uses **Tiny-ImageNet-200**. To simulate data scarcity, we sample
**class-balanced** subsets across all 200 classes:

| Split | Images | Purpose |
|---|---|---|
| `1000`  | ~1,000  | Extreme scarcity |
| `5000`  | ~5,000  | Low data |
| `10000` | ~10,000 | Moderate data |
| `25000` | ~25,000 | Relatively data-rich |

The **validation set is held fixed (full)** across every split to keep evaluation fair.
Splits are reproducibly generated by [`src/make_splits.py`](src/make_splits.py).

### Loss Functions Compared

| Loss | File | Notes |
|---|---|---|
| **Vanilla GAN** (BCE) | [`losses/vanilla_gan.py`](losses/vanilla_gan.py) | Original min-max with `BCEWithLogitsLoss` |
| **LSGAN** | [`losses/lsgan.py`](losses/lsgan.py) | Least-squares objective |
| **WGAN** | [`losses/wgan.py`](losses/wgan.py) | Wasserstein critic + weight clipping `[-0.01, 0.01]` |
| **WGAN-GP** | [`losses/wgan_gp.py`](losses/wgan_gp.py) | Wasserstein + gradient penalty (`λ = 10`) |

## Key Findings

> Metrics below come directly from the training logs in `experiments/logs/`.
> **Training stability** is quantified as the **standard deviation of the generator
> loss** over training (lower = more stable).

### 1. LSGAN is the most consistently stable objective
LSGAN produced the **smoothest, lowest-variance** training across *every* dataset size,
and its stability improved monotonically as more data became available:

| Dataset size | 1000 | 5000 | 10000 | 25000 |
|---|---|---|---|---|
| **LSGAN** generator-loss std | 0.886 | 0.265 | 0.129 | **0.075** |

### 2. WGAN (weight clipping) collapses under extreme scarcity
WGAN was **stable once enough data was present** (std ≈ 0.03–0.08 at 5k–25k), but at the
**1,000-image** regime it diverged catastrophically (critic loss ≈ −5392, generator-loss
std ≈ 918). This illustrates the well-known fragility of **weight clipping** when the
critic sees too few real samples.

### 3. Vanilla GAN is moderate but exhibits classic saturation
The BCE objective stayed numerically stable but settled at **persistently high generator
loss (≈ 5–6)** with moderate variance — the textbook saturating-gradient behaviour, only
partially relieved by more data.

### 4. WGAN-GP is the most expensive — and cost grows with data
The gradient penalty plus `n_critic = 5` made WGAN-GP **dramatically more compute-heavy**.
Average epoch time scaled steeply with dataset size, far beyond the other objectives:

| Loss | Epoch time @ 25k (s) |
|---|---|
| Vanilla | ~26 |
| LSGAN | ~27 |
| WGAN | ~62 |
| **WGAN-GP** | **~557** |

Its generator loss also operated at a much larger magnitude (and higher variance),
reflecting the unbounded critic typical of the Wasserstein-GP formulation.

### Takeaway
- **Very low data (≤1k):** prefer **LSGAN**; avoid plain **WGAN** (clipping collapse).
- **Stability per unit compute:** **LSGAN** offers the best trade-off.
- **WGAN-GP:** strong regularization but a heavy compute tax that scales poorly here.
- **No universal winner** — the optimal objective is regime-dependent.

## Reproducing the Study

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Build the class-balanced scarcity splits from raw Tiny-ImageNet
python src/make_splits.py

# 3. Run a single experiment (loss × dataset size)
#    Edit the call at the bottom of training/run_experiment.py, e.g.:
#    train(loss_name="lsgan", split_size=5000, epochs=50, batch_size=32)
python training/run_experiment.py

# 4. Regenerate the publication figures from the logs
python notebooks/research_plots.py
```

Each run writes a CSV log (`epoch, d_loss, g_loss, epoch_time, params_G, params_D`),
periodic sample grids, and final `G.pth` / `D.pth` checkpoints under `experiments/`.

**Loss options:** `vanilla` · `lsgan` · `wgan` · `wgan_gp`
**Dataset sizes:** `1000` · `5000` · `10000` · `25000`
